# v0.6.0 — Mobile Android (Play Store)

**Data:** 2026-06-11
**Onda:** 6

## Resumo

A Orca ganha sua primeira versão **nativa para Android**, empacotada com Capacitor e pronta para distribuição na Google Play Store. Pagamentos no app passam a usar **Google Play Billing** via **RevenueCat**, mantendo o Stripe na web. Web e mobile compartilham o mesmo banco de entitlements — quem assina no app vê o plano ativo no site, e vice-versa.

## Novidades para o usuário

- 📱 **App Android nativo** — instala da Play Store, ícone próprio, splash com a orquinha.
- 💳 **Assinatura no app** — comprar Pro/Business direto no celular via Google Play Billing.
- 🔄 **Plano sincronizado** — assinou no celular, libera no site. Cancela em um, cancela no outro.
- 🔗 **Deep links** — abrir um link de proposta (`orca-mento.app/p/...`) abre direto no app instalado.

## Mudanças técnicas

### Adicionado
- `capacitor.config.ts` na raiz (`appId: app.orca.mento`, `appName: Orca`).
- Dependências: `@capacitor/core`, `@capacitor/android`, `@capacitor/app`, `@capacitor/status-bar`, `@capacitor/splash-screen`, `@revenuecat/purchases-capacitor`.
- `src/lib/platform.ts` — helpers `isNativeMobile()`, `isAndroidNative()`.
- `src/hooks/useRevenueCat.ts` — bootstrap RC + login com `user.id`.
- `src/pages/MobilePaywall.tsx` — paywall nativo, substitui `/pricing` no app.
- Edge function `supabase/functions/revenuecat-webhook` — auditoria + reconciliação de `user_plans`.
- Tabela `revenuecat_events` (admin-only) para auditoria do webhook.
- Coluna `plans.metadata` (JSON) com `rc_product_id` para mapear plano ↔ produto Play Console.
- `public/.well-known/assetlinks.json` (placeholder do SHA-256 do certificado).

### Alterado
- `src/main.tsx` — bootstrap de `StatusBar` e `SplashScreen` em native (no-op na web).
- `src/App.tsx` — rota `/pricing` serve `MobilePaywall` em native.
- `supabase/config.toml` — registra `revenuecat-webhook` com `verify_jwt = false`.

## Configuração necessária (operacional)

### Conta Google Play Console
1. Registrar como **organização** com CNPJ do MEI (US$ 25 único).
2. Aguardar verificação D-U-N-S (gratuita, ~14 dias).
3. Criar app `Orca` com `applicationId = app.orca.mento`.

### Produtos no Play Console
Em **Monetize → Subscriptions** criar:
- `orca_pro_monthly`, `orca_pro_yearly`
- `orca_business_monthly`, `orca_business_yearly`

Preços em BRL. Salvar cada `product_id` no plano correspondente, em `plans.metadata->>'rc_product_id'`.

### Conta RevenueCat
1. Criar projeto Orca, app Android.
2. Conectar Play Console (service account JSON).
3. Configurar Offerings com os 4 packages.
4. Webhook → `https://<projeto>.supabase.co/functions/v1/revenuecat-webhook`
   - Authorization header: `Bearer <valor de RC_WEBHOOK_SECRET>`

### Secrets / env vars
- **Runtime (Lovable Cloud)**: `RC_WEBHOOK_SECRET` (validação do webhook).
- **Build-time (.env do build Android)**: `VITE_REVENUECAT_ANDROID_KEY` (chave pública do SDK).

### Build local (você roda)
```bash
npm install
npx cap add android
npx cap sync
npx cap open android   # gera .aab no Android Studio
```

Gerar `keystore` no Android Studio para assinar releases — **guardar com a vida**.

### Asset Links (deep linking)
Substituir `REPLACE_WITH_SHA256_FROM_PLAY_CONSOLE_APP_SIGNING` em `public/.well-known/assetlinks.json` pelo SHA-256 do certificado de assinatura do app (Play Console → Setup → App integrity).

## Breaking changes / Migrações

- Migração SQL: adiciona `plans.metadata` (default `{}`) e cria `revenuecat_events`. Nenhuma quebra de comportamento existente.
- Web continua usando Stripe sem nenhuma mudança.

## Métricas e limites afetados
